# SDCCD SSL Tools

Application to manage various tasks for SSL and CSRs without needing to use IIS.

There are four main features provided by command line argument verbs:

- create
- finish
- autofix
- info

## Create Verb

Use the _create_ verb to create a CSR and key file. The files will be saved in the current working directory.

```bash
ssl-tools.exe help create
  -c, --cn         Common Name for the CSR (required)
  -s, --san        Subject Alternative Name (seperate multiple values with a space, optional)
  -l, --keysize    Key Size to use (default 4096 but can be 2048/3072/4096, optional)
  -k, --key        External private key to use (optional)
Examples:
#Basic CSR with an autogenerated key of 4096 bits
ssl-tools.exe create --cn some-domain.org --san www.some-domain.org
#The same command but with short options
ssl-tools.exe create -c some-domain.org -s www.some-domain.org
#CSR with an autogenerated key of 2048 bits
ssl-tools.exe create --cn some-domain.org --san www.some-domain.org --keysize 2048
#CSR with an external key file (bit size depends on the key in file)
ssl-tools.exe create --cn some-domain.org --san www.some-domain.org --key mykeyfile.key
#CSR with multiple SAN values
ssl-tools.exe create --cn some-domain.org --san www.some-domain.org mail.some-domain.org api.some-domain.org
```
Even when you do not pass any SAN values the Common Name gets added as a DNS SAN. This generally makes installing the certificates easier with the various systems that need them.

The CSR is created with these Basic Constraints:
* Certificate Authority = false
* HasPathLengthConstraint = false
* PathLengthConstraint = 0
* Critical = false

Additionally, the key usage extensions are
* Digital Signature
* Key Encipherment

## Finish Verb

Use the _finish_ verb to complete a CSR by combining a signed certificate and the key file from a previous create step.

```bash
ssl-tools.exe help finish
  -c, --cert    PEM encoded certificate file from CA (required)
  -k, --key     Key file from a previous 'Create CSR' action (required)
  -p, --pfx     PFX file to create. Defaults to the name of the certificate minus file extension if not included (optional)
  --password
                The password to use to protect the PFX file (required).
                If not passed as a command line argument then the environment variable 'sslpass' is checked.
  --chain       Include the certificate chain in the PFX file (optional)
  --include-root
                Include the root certificate as part of the PFX chain (optional)
Example:
#Basic command to create a PFX
ssl-tools.exe finish --cert cert-from-ca.pem --key some-domain.org.key --pfx cert-to-create.pfx --password s3cuReP4s$
#Create a full-chain PFX using the password in the environment variable 'sslpass'
ssl-tools.exe finish -c cert-from-ca.pem -k some-domain.org.key --chain --include-root
```

When using the --chain switch the application implicitly calls the "Autofix" method on the source certificate as a first step. This is done to ensure that a valid chain is present to export. Please see the autofix notes for details on that process.

## Autofix Verb

Use the _autofix_ verb to automatically fix the order of a certificate chain without providing any user input.

```bash
ssl-tools.exe help autofix
  -c, --cert    The certificate to fix (required)
  -o, --oracle  Prepare additional certificates for Oracle (optional)
  -b, --bypass-local-validation
                Do not use local system stores to validate CA certificates exist (optional)
  -z, --zip     Zip resulting certificates (optional)
  -d, --der     Save certificate in binary DER format (optional)
Examples:
#Will correct the order of the certficate chain from InCommon for *.dev.sdccd.edu
ssl-tools.exe autofix --cert _dev.sdccd.edu_chain.pem
#Will correct the order of the certificate chain from InCommon for csdev.dev.sdccd.edu
#and also generate the three certificate files needed to install on Oracle/PeopleSoft systems
ssl-tools.exe autofix -c csdev.dev.sdccd.edu.pem -o
#Will correct the order of the certificate chain and create a new file for the binary certificate chain.
#The resulting output file will have a .der file extension
ssl-tools.exe autofix -c csdev.dev.sdccd.edu.pem -d
```

#### Note on autofix with single subject certificate file
If you run autofix on a certificate file that does not contain a chain (for example, only a subject certificate) the application will attempt to build the chain using the [Windows Certificate Chain Engine rules](https://www.sysadmins.lv/blog-en/certificate-chaining-engine-how-this-works.aspx). The single certificate file will be updated/overwritten with the new certificate chain

#### Creating Oracle certificates
When using the --oracle switch an additional four files get created:
- csdev.sdccd.edu_oracle__client.pem
- csdev.sdccd.edu_oracle__intermediate.pem
- csdev.sdccd.edu_oracle__root.pem
- csdev.sdccd.edu_oracle___fullchain.pem

These are all files that normally get manually created as part of the certificate installation process for PeopleSoft.
The original certificate chain that was autofixed by the process is suitable for use with the OCI load balancers, but only if the --der flag is not used.

#### Bypassing local validation
Use this switch when the certificate bundle includes intermediate certificates that are marked as certificate authorities but are not installed on the host system. By default any intermediate certificate is trusted implicitly when part of a bundle and properly signed by a trusted Certificate Authority. But local validation treats each certificate as if it were isolated from the bundle. If a certificate includes a Basic Constraint Extension of _CA:true_ the local validation expects it to be installed on the system as a trusted CA certificate.

#### Zip file creation
When using the zip switch will create a password protected zip file. The password used is read from an environment variable named _sslpass_. If not found then the default value of _SSL-PASSWORD_ is used.
Any file that matches either the source file name or the subject certificate's Common Name will be included in the zip file (except for any CSR files).


## Info Verb

Use the _info_ verb to view details about a certificate or certificate request.

```bash
ssl-tools.exe help info
  -c, --cert    The certificate(s) to view (optional)
  -r, --csr     The CSR to view (optional)
  -u, --url     The URL(s) to fetch certificate and chain (optional)
  -s, --short   Show short summary only (optional)
Examples:
#Will show information about a csr
ssl-tools.exe info --csr _dev.sdccd.edu.csr
#Will show information about a certificate
ssl-tools.exe info --cert _dev.sdccd.edu.pem
#You can include both --cert and --csr
ssl-tools.exe info --cert _dev.sdccd.edu.pem --csr _dev.sdccd.edu.csr
#You can include multiple certificates, seperated by spaces
ssl-tools.exe info -c _dev.sdccd.edu.pem _test.sdccd.edu.pem
#You can also include multiple URLs, seperated by spaces
ssl-tools.exe info -u https://myportal.sdccd.edu https://www.sdccd.edu
```
